Callback Functions

  • Uma callback é uma função que é passada como argument para outra função
  • Essa técnica permite com que uma função possa chamar outra
  • Callbacks se destacam quando usadas em conjunto com funções assíncronas
  • É importante relembrar que a função callback é PASSADA

Exemplo

                        
function myDisplayer(some) {
document.getElementById("demo").innerHTML = some;
}

function myCalculator(num1, num2, myCallback) {
let sum = num1 + num2;
myCallback(sum);
}

myCalculator(5, 5, myDisplayer); ← the function being PASSED as an argument ( important to remember that we do not use parenthesis with callbacks)
  • Callbacks normalmente são usadas junto com funções assíncronas
  • Exemplo:
                        
setTimeout(sendMessage, 3000);  → setTimeout is a asynchronous function

function sendMessage() {
document.getElementById("demo").innerHTML = "Hello World!";
}
  • Ao invés de passar apenas o nome da função como um argument, podemos passar a função inteira
  • Exemplo:
                        
setTimeout(function() { sendMessage("Hello World!"); }, 3000);

function sendMessage(value) {
    document.getElementById("demo").innerHTML = value;
}

Funções são executadas na ordem em que elas são chamadas, e não na ordem em que são definidas.

Aguardando por Intervalos

  • Usando a função setInterval() podemos especificar uma função callback para ser executada a cada intervalo de tempo
  • Exemplo:
                            
setInterval(myFunction, 1000);

function myFunction() {
    let d = new Date();
        document.getElementById("demo").innerHTML= d.getSeconds();
}

*Nesse exemplo, "myFunction" é usada como callback, sendo chamada novamente a cada 1 segundo = 1000 milissegundos

  • Uma função carregando um recurso externo (como um script ou um arquivo), não poderá usar o conteúdo antes dele estar totalmente carregado.

O que é Fetch?

Fetch é a recuperação de dados por um script, software ou até mesmo um hardware.

Após serem recuperados, os dados podem ser exibidos em uma tela ou movidos para um local alternativo.

Loop for ... in

O Loop for ... in faz a iteração sobre todas as propriedades enumeráveis de um object
O object deve conter keys em formato String

GET e POST

GET Request

  • É mais simples e mais rápido, podendo ser usando na maioria dos casos
  • Exemplo:
                            
xhttp.open("GET", "demo_get.asp");
xhttp.send();
  • Caso queira enviar informações com o método GET, adicione a informação na URL
  • Exemplo:
                            
xhttp.open("GET", "demo_get2.asp?fname=Henry&lname=Ford");
xhttp.send();

POST Request

POST requests devem ser usadas quando:

  • Um arquivo em cache não é uma opção;
    • Atualizar um arquivo ou banco de dados no servidor.
  • Enviando uma quantidade grande de dados ao servidor;
    • POST não tem limite de tamanho de dados.
  • Enviando inputs do usuário;
    • POST é mais robusto e seguro.
  • Exemplo:
                            
xhttp.open("POST", "demo_post.asp");
xhttp.send();

GET e SET

  • GET
    • Pega uma informação do servidor
  • SET
    • Envia/registra uma informação para o servidor

Javascript Hoisting

  • Hoisting é um comportamento padrão do JavaScript.
  • Hoisting é reponsável por mover as declarações para o topo do escopo atual.
  • O hoisting aplica-se para a declaração de variáveis e também funções
    • Variáveis Javascript podem ser chamadas antes de serem declaradas
    • Funções JavaScript podem ser chamadas antes de serem declaradas
    • Funções definidas em uma expressão não são consideradas como hoisting

Propriedade de um Object e Método do Object

Propriedade de um Object

  • Uma função definida como PROPRIEDADE DE UM OBJECT é chamada de MÉTODO DO OBJETO

Método de um Object

  • Uma função feita para CRIAR NOVOS OBJECTS é chamada de OBJECT CONSTRUCTOR

Propriedades do input text

readOnly

  • Um campo read-only não pode ser modificado, contudo o usuário pode:
    • Navegar até o campo usando TAB;
    • Destacar o campo;
    • Copiar o texto contido no campo.
  • Input Fields com a propriedade readOnly ainda são enviados pelo formulário;
  • Pode ser definido como true ou false.
  • Incluindo readOnly diretamente no <input>
  • Exemplo:
                            
<input type="text" id="myText" readOnly>
  • Incluindo readOnly através de uma função
  • Exemplo:
                            
function myFunction() {
    document.getElementById("myText").readOnly = true;
}
  • Verificando se um <input> é readOnly
  • Exemplo:
                            
var isReadOnly = document.getElementById("myText").readOnly;
if ( isReadOnly == true )
    console.log( "É readOnly" );
else
    console.log( "Não é readOnly" );

disabled

  • Um elemento com a propriedade disabled:
    • É inutilizável;
    • Não pode ser clicado.
  • Input Fields com a propriedade disabled não são enviados pelo formulário;
  • Pode ser definido como true ou false.
  • Normalmente são renderizados na cor cinza pelos browsers.
  • Incluindo disabled diretamente no <input>
  • Exemplo:
                            
<input type="text" id="myText" disabled>
  • Incluindo disabled através de uma função
  • Exemplo:
                            
function myFunction() {
    document.getElementById("myText").disabled = true;
}
  • Verificando se um <input> possui a propriedade disabled
  • Exemplo:
                            
var isDisabled = document.getElementById("myText").disabled;
if ( isDisabled == true )
    console.log( "Está desabilitado" );
else
    console.log( "Não está desabilitado" );
